# Planner — Мобильный Ежедневник

Planner — это современное Native Android приложение для управления задачами, разработанное в рамках тестового задания. Приложение построено на принципах **Clean Architecture** с использованием **MVI (Model-View-Intent)** и самого современного стека технологий.

## 🚀 Стек технологий

* **Язык:** Kotlin 2.3.20
* **UI:** Jetpack Compose + Material 3 (адаптивная верстка, поддержка темной темы).
* **Архитектура:** Clean Architecture + MVI.
* **DI:** Hilt.
* **База данных:** Room (локальное хранение задач).
* **Асинхронность:** Coroutines & Flow.
* **Навигация:** Jetpack Compose Navigation (Type-Safe).
* **Сериализация:** Kotlinx Serialization.
* **Тестирование:** Unit-тесты (MockK, Turbine, Coroutines Test).

## 🏗 Архитектура

Проект разделен на слои согласно принципам чистой архитектуры для обеспечения высокой тестируемости и независимости компонентов:

1. **Data Layer:** Реализация репозиториев, работа с Room DAO и сервис парсинга локального JSON.
2. **Domain Layer (UseCases):** Изолированная бизнес-логика. Каждый UseCase отвечает за одну конкретную операцию (например, сохранение задачи или получение списка на день).
3. **UI Layer (Presentation):** Реализован на паттерне MVI. ViewModels управляют состоянием (`State`), обрабатывают намерения пользователя (`Intent`) и отправляют одноразовые события (`Effect`).

## 🏗 Архитектура проекта

Проект разделен на слои для обеспечения масштабируемости и тестируемости:

- `:app` — Точка входа, конфигурация Hilt, навигационный граф.
- `:core` — Общие компоненты (`:mvi`, `:ui`, `:navigation`, `:common`).
- `:data` — Слой данных (Room Database, Repository, JSON Parser).
- `:feature` — Независимые фичи (`task-list`, `task-editor`), разделенные на `:api` и `:impl`.

  ## 💾 Формат данных

Данные хранятся в формате, совместимом с требуемым JSON-структурой. Поля времени используют `timestamp (Long)`.

**Пример объекта:**
```json
{
  "id": 1,
  "date_start": "1776076800000",
  "date_finish": "1776080400000",
  "name": "Собеседование",
  "description": "Обсуждение технических деталей проекта и архитектуры."
}
```

## 🌟 Основные возможности

* **Календарь:** Интерактивный заголовок с возможностью выбора даты. Адаптивное сворачивание при прокрутке списка.
* **Список задач:** Группировка задач по часам (формат 14.00-15.00).
* **Редактор:** Экран создания/редактирования с валидацией данных и выбором времени через Material 3 TimePicker.
* **Локальное хранилище:** Автоматическое заполнение базы данных из `tasks.json` при первом запуске.
* **Адаптивность:** Поддержка портретной и альбомной ориентации (разделенный экран в Landscape).
* **Производительность:** Группировка и форматирование данных оптимизированы с помощью `remember` и современного `java.time` API.

## 🛠 Установка и запуск

1. Клонируйте репозиторий.
2. Откройте проект в **Android Studio Ladybug** (или новее).
3. Убедитесь, что установлен JDK 17+.
4. Соберите проект: `./gradlew assembleDebug`.
5. Запуск Unit-тестов: `./gradlew test`.

## 🧪 Тестирование

Проект покрыт Unit-тестами, проверяющими:

* Маппинг данных в UseCase-ах.
* Переходы состояний и обработку Side-effects во ViewModels.
* Логику инициализации данных в репозитории.

Используются библиотеки `MockK` для мокирования зависимостей и `Turbine` для тестирования потоков данных (Flow).
